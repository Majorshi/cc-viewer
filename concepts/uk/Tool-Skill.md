# Skill

Викликає іменовану навичку (skill) всередині поточної розмови. Навички — це попередньо спаковані пакети можливостей — доменні знання, робочі процеси та іноді доступ до інструментів — які оболонка надає асистенту через системні нагадування.

## Коли використовувати

- Користувач вводить слеш-команду, таку як `/review` чи `/init` — слеш-команди є навичками і повинні виконуватися через цей інструмент.
- Користувач описує завдання, що відповідає умовам активації оголошеної навички (наприклад, запит сканувати транскрипти на повторювані запити дозволу відповідає `fewer-permission-prompts`).
- Заявлена мета навички є прямим збігом з поточним файлом, запитом чи контекстом розмови.
- Доступні спеціалізовані, повторювані робочі процеси у вигляді навичок, і канонічна процедура є кращою за ad-hoc підхід.
- Користувач запитує "які навички доступні" — перелічіть оголошені імена та викликайте лише після підтвердження.

## Параметри

- `skill` (string, обовʼязковий): Точне імʼя навички, перелічене в поточному системному нагадуванні доступних навичок. Для навичок з простору імен плагіна використовуйте повністю кваліфіковану форму `plugin:skill` (наприклад, `skill-creator:skill-creator`). Не включайте провідну косу риску.
- `args` (string, необовʼязковий): Довільні аргументи, передані навичці. Формат і семантика визначаються документацією кожної навички.

## Приклади

### Приклад 1: Запуск навички огляду на поточній гілці

```
Skill(skill="review")
```

Навичка `review` пакує кроки для огляду pull request відносно поточної базової гілки. Її виклик завантажує визначену оболонкою процедуру огляду в поточний крок.

### Приклад 2: Виклик навички з простору імен плагіна з аргументами

```
Skill(
  skill="skill-creator:skill-creator",
  args="create a skill that summarizes git log for a given date range"
)
```

Маршрутизує запит через точку входу плагіна `skill-creator`, щоб запустився робочий процес створення.

## Примітки

- Викликайте лише навички, імена яких зʼявляються буквально в системному нагадуванні доступних навичок, або навички, які користувач ввів безпосередньо як `/name` у своєму повідомленні. Ніколи не вгадуйте та не вигадуйте імена навичок з памʼяті чи даних навчання — якщо навичка не оголошена, не викликайте цей інструмент.
- Коли запит користувача відповідає оголошеній навичці, виклик `Skill` є блокуючою передумовою: викликайте її перед генерацією будь-якої іншої відповіді про завдання. Не описуйте, що робитиме навичка — запускайте її.
- Ніколи не згадуйте навичку за імʼям, не викликаючи її насправді. Оголошення навички без виклику інструмента вводить в оману.
- Не використовуйте `Skill` для вбудованих команд CLI, таких як `/help`, `/clear`, `/model` чи `/exit`. Вони обробляються оболонкою безпосередньо.
- Не викликайте повторно навичку, яка вже виконується в поточному кроці. Якщо ви бачите тег `<command-name>` у поточному кроці, навичка вже завантажена — виконуйте її інструкції на місці замість повторного виклику інструмента.
- Якщо кілька навичок могли б застосуватися, оберіть найспецифічнішу. Для змін конфігурації, таких як додавання дозволів чи хуків, віддавайте перевагу `update-config` над загальним підходом до налаштувань.
- Виконання навички може вводити нові системні нагадування, інструменти чи обмеження до кінця кроку. Перечитайте стан розмови після завершення навички перед продовженням.
