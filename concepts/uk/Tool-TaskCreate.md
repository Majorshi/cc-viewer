# TaskCreate

Створює нове завдання в списку завдань поточної команди (або в списку завдань сесії, коли команда не активна). Використовуйте для захоплення робочих пунктів, які слід відстежувати, делегувати чи повертатися до них пізніше.

## Коли використовувати

- Користувач описує багатоетапну роботу, що виграє від явного відстеження.
- Ви розбиваєте великий запит на менші, окремо завершувані одиниці.
- Продовження виявлено посеред завдання і його не слід забути.
- Вам потрібен стійкий запис наміру перед передачею роботи члену команди чи підагенту.
- Ви працюєте в режимі плану і хочете, щоб кожен крок плану був представлений як конкретне завдання.

Пропустіть `TaskCreate` для тривіальних одноразових дій, чистої розмови чи будь-чого, що можна виконати за два-три прямі виклики інструментів.

## Параметри

- `subject` (string, обовʼязковий): Короткий імперативний заголовок, наприклад `Fix login redirect on Safari`. Тримайте його приблизно до восьмидесяти символів.
- `description` (string, обовʼязковий): Детальний контекст — проблема, обмеження, критерії прийому та будь-які файли чи посилання, які знадобляться майбутньому читачеві. Пишіть так, наче член команди візьметься за це з нуля.
- `activeForm` (string, необовʼязковий): Текст індикатора в теперішньому тривалому часі, показаний, поки завдання має статус `in_progress`, наприклад `Fixing login redirect on Safari`. Повторіть `subject`, але у формі на -ing.
- `metadata` (object, необовʼязковий): Довільні структуровані дані, прикріплені до завдання. Типові застосування: мітки, підказки пріоритету, ID зовнішніх тикетів чи специфічна для агента конфігурація.

Новостворені завдання завжди починаються зі статусом `pending` без власника. Залежності (`blocks`, `blockedBy`) не встановлюються під час створення — застосовуйте їх пізніше через `TaskUpdate`.

## Приклади

### Приклад 1

Захоплення повідомлення про помилку, щойно поданого користувачем.

```
TaskCreate(
  subject: "Repair broken PDF export on Windows",
  description: "Users on Windows 11 report the export button produces a 0-byte file. Reproduce with sample doc in test/fixtures/export/, then fix the code path in src/export/pdf.ts. Acceptance: export writes a valid PDF and the existing export test suite passes.",
  activeForm: "Repairing broken PDF export on Windows"
)
```

### Приклад 2

Розділіть епік на відстежувані одиниці на початку сесії.

```
TaskCreate(
  subject: "Draft migration plan for auth service",
  description: "Produce a written plan covering rollout stages, rollback strategy, and monitoring. Output: docs/auth-migration.md.",
  activeForm: "Drafting migration plan for auth service",
  metadata: { "priority": "P1", "linearId": "AUTH-214" }
)
```

## Примітки

- Пишіть `subject` в імперативному стилі, а `activeForm` у теперішньому тривалому, щоб UI читався природно, коли завдання переходить у `in_progress`.
- Викликайте `TaskList` перед створенням, щоб уникнути дублікатів — список команди спільний з членами команди та підагентами.
- Не включайте секрети чи облікові дані в `description` чи `metadata`; записи завдань видимі кожному, хто має доступ до команди.
- Після створення перемещуйте завдання через його життєвий цикл через `TaskUpdate`. Не залишайте роботу мовчки покинутою в `in_progress`.
