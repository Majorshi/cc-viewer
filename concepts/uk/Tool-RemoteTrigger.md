# RemoteTrigger

Викликає API віддалених тригерів claude.ai для керування запланованими завданнями та виконанням тригерів на вимогу. Токен OAuth обробляється інструментом внутрішньо і ніколи не передається моделі або оболонці.

## Коли використовувати

- Керування віддаленими агентами (тригерами) на claude.ai, включаючи перегляд списку, перевірку та оновлення існуючих тригерів
- Створення нового автоматизованого завдання на основі cron, яке запускає агента Claude за повторюваним розкладом
- Запуск існуючого тригера на вимогу без очікування наступного запланованого виконання
- Відображення або аудит усіх поточних тригерів для перегляду їхньої конфігурації та стану
- Оновлення налаштувань тригера, таких як розклад, корисне навантаження або опис, без необхідності його повторного створення

## Параметри

- `action` (string, обов'язковий): операція, яку необхідно виконати — одне з значень `list`, `get`, `create`, `update` або `run`
- `trigger_id` (string, обов'язковий для `get`, `update` та `run`): ідентифікатор тригера, над яким виконується операція; повинен відповідати шаблону `^[\w-]+$` (лише символи слів і дефіси)
- `body` (object, обов'язковий для `create` і `update`; необов'язковий для `run`): корисне навантаження запиту, що надсилається до API

## Приклади

### Приклад 1: отримати список усіх тригерів

```json
{
  "action": "list"
}
```

Викликає `GET /v1/code/triggers` і повертає масив JSON усіх тригерів, пов'язаних з аутентифікованим обліковим записом.

### Приклад 2: створити новий тригер, що запускається кожного ранку в робочий день

```json
{
  "action": "create",
  "body": {
    "name": "weekday-morning-report",
    "schedule": "0 8 * * 1-5",
    "description": "Генерувати щоденний підсумок кожного робочого дня о 08:00 UTC"
  }
}
```

Викликає `POST /v1/code/triggers` із наданим тілом запиту і повертає щойно створений об'єкт тригера, включаючи призначений `trigger_id`.

### Приклад 3: запустити тригер на вимогу

```json
{
  "action": "run",
  "trigger_id": "my-report-trigger"
}
```

Негайно викликає `POST /v1/code/triggers/my-report-trigger/run`, обминаючи запланований час.

### Приклад 4: отримати один тригер

```json
{
  "action": "get",
  "trigger_id": "my-report-trigger"
}
```

Викликає `GET /v1/code/triggers/my-report-trigger` і повертає повну конфігурацію тригера.

## Примітки

- Токен OAuth вводиться інструментом усередині процесу — ніколи не копіюйте, не вставляйте і не записуйте токени вручну; це створює загрозу безпеці і є зайвим при використанні цього інструменту.
- Використовуйте цей інструмент замість прямого `curl` або інших HTTP-клієнтів для всіх викликів API тригерів; пряме використання HTTP обходить безпечне введення токена і може розкрити облікові дані.
- Інструмент повертає необроблену відповідь JSON від API; викликаюча сторона відповідає за розбір відповіді та обробку кодів стану помилок.
- Значення `trigger_id` повинно відповідати шаблону `^[\w-]+$` — дозволені лише буквено-цифрові символи, підкреслення та дефіси; пробіли або спеціальні символи призведуть до помилки запиту.
