# executeCode

Виконує фрагмент коду всередині живого ядра чи пісочниці, наданої інтеграцією IDE (наприклад, Jupyter-ядра, привʼязаного до поточно відкритого ноутбука). Інструмент присутній лише тоді, коли Claude Code працює поряд із сумісним мостом IDE, таким як розширення VS Code із вибраним Jupyter-ядром.

## Коли використовувати

- Швидке обчислення, перевірка даних чи побудова графіка на стані, вже завантаженому в активному Jupyter-ядрі.
- Валідація фрагмента коду перед вставкою його в клітинку ноутбука.
- Дослідження dataframe у памʼяті, змінної чи моделі, що існує в ядрі, але не серіалізована на диск.
- Створення графіка чи числового результату, який користувач хоче відрендерити вбудовано в IDE.

НЕ використовуйте для самостійних скриптів, які краще обслуговуватиме `Bash`, що запускає `python script.py`, або для коду, що потребує збереження між свіжими ядрами.

## Параметри

- `code` (string, обовʼязковий): Код для виконання в поточному ядрі. Виконується так, наче вставлений у клітинку ноутбука — змінні, визначені, зберігаються в ядрі, поки його не перезапустять.
- `language` (string, необовʼязковий): Мова фрагмента, коли міст IDE підтримує кілька ядер. Найчастіше опускається; за замовчуванням — мова активного ядра (зазвичай Python).

## Приклади

### Приклад 1: Перевірка dataframe у памʼяті

```
executeCode(
  code="df.head()\nprint(df.shape)\nprint(df.dtypes)"
)
```

Повертає перші рядки, форму та типи колонок dataframe, уже завантаженого в ядрі.

### Приклад 2: Швидка числова перевірка

```
executeCode(
  code="import numpy as np\nnp.mean([1, 2, 3, 4, 5])"
)
```

Виконує одноразове обчислення без створення клітинки ноутбука.

## Примітки

- `executeCode` — інструмент моста IDE. Він недоступний у простих термінальних сесіях Claude Code; він зʼявляється лише тоді, коли сесія підключена до IDE, що експонує ядро (наприклад, розширення Jupyter VS Code).
- Стан зберігається в ядрі. Змінні, визначені одним викликом `executeCode`, залишаються видимими для пізніших викликів, клітинок ноутбука та користувача — уважно ставтеся до побічних ефектів.
- Тривалий чи блокуючий код блокуватиме ядро. Тримайте фрагменти короткими; для багатохвилинної роботи пишіть справжній скрипт і запускайте його через `Bash`.
- Вивід (stdout, значення, що повертаються, зображення) повертається в сесію. Дуже великі виводи можуть бути обрізані мостом IDE.
- Для правок файлів віддавайте перевагу `Edit`, `Write` чи `NotebookEdit` — `executeCode` не замінює авторства файлів вихідного коду.
