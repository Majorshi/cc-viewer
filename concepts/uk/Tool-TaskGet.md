# TaskGet

Отримує повний запис одного завдання за ID, включно з його описом, поточним статусом, власником, метаданими та ребрами залежностей. Використовуйте, коли підсумок, повернутий `TaskList`, недостатній для дії із завданням.

## Коли використовувати

- Ви вибрали завдання з `TaskList` і потребуєте повного опису перед початком роботи.
- Ви збираєтеся позначити завдання як `completed` і хочете перевірити критерії прийому ще раз.
- Вам потрібно перевірити, які завдання це блокує (`blocks`) або ким воно заблоковане (`blockedBy`), щоб вирішити наступний крок.
- Ви досліджуєте історію — хто ним володіє, які метадані були прикріплені, коли воно змінювало стан.
- Член команди чи попередня сесія посилалися на ID завдання, і вам потрібен контекст.

Віддавайте перевагу `TaskList`, коли вам потрібен лише високорівневий огляд; резервуйте `TaskGet` для конкретного запису, який ви маєте намір уважно прочитати чи модифікувати.

## Параметри

- `taskId` (string, обовʼязковий): Ідентифікатор завдання, повернутий `TaskCreate` чи `TaskList`. ID стабільні протягом усього життя завдання.

## Приклади

### Приклад 1

Перегляд завдання, яке ви щойно побачили в списку.

```
TaskGet(taskId: "t_01HXYZ...")
```

Типові поля відповіді: `id`, `subject`, `description`, `activeForm`, `status`, `owner`, `blocks`, `blockedBy`, `metadata`, `createdAt`, `updatedAt`.

### Приклад 2

Розвʼяжіть залежності перед початком.

```
TaskGet(taskId: "t_01HXYZ...")
# Перевірте blockedBy — якщо будь-яке посилане завдання все ще pending
# або in_progress, спочатку працюйте над блокатором.
```

## Примітки

- `TaskGet` лише для читання і безпечний для багаторазового виклику; він не змінює статус чи власника.
- Якщо `blockedBy` не порожній і містить завдання, які не `completed`, не починайте це завдання — спочатку розвʼяжіть блокатори (або скоординуйтеся з їхнім власником).
- Поле `description` може бути довгим. Прочитайте його повністю перед діями; перегляд по діагоналі призводить до пропущених критеріїв прийому.
- Невідомий чи видалений `taskId` повертає помилку. Перезапустіть `TaskList`, щоб вибрати поточний ID.
- Якщо ви збираєтеся редагувати завдання, спочатку викличте `TaskGet`, щоб уникнути перезапису полів, які член команди щойно змінив.
