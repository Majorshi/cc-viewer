# RemoteTrigger

เรียกใช้ API ทริกเกอร์ระยะไกลของ claude.ai เพื่อจัดการงานที่กำหนดเวลาไว้และการเรียกใช้ทริกเกอร์ตามต้องการ โทเค็น OAuth ได้รับการจัดการภายในโดยเครื่องมือและไม่เคยถูกเปิดเผยต่อโมเดลหรือเชลล์

## เมื่อใดควรใช้

- จัดการเอเจนต์ระยะไกล (ทริกเกอร์) บน claude.ai รวมถึงการแสดงรายการ ตรวจสอบ และอัปเดตทริกเกอร์ที่มีอยู่
- สร้างงานอัตโนมัติแบบ cron ใหม่ที่รัน Claude agent ตามกำหนดการที่เกิดซ้ำ
- เรียกใช้ทริกเกอร์ที่มีอยู่ตามต้องการโดยไม่ต้องรอการรันตามกำหนดการครั้งถัดไป
- แสดงรายการหรือตรวจสอบทริกเกอร์ปัจจุบันทั้งหมดเพื่อดูการกำหนดค่าและสถานะ
- อัปเดตการตั้งค่าทริกเกอร์ เช่น กำหนดการ เพย์โหลด หรือคำอธิบาย โดยไม่ต้องสร้างทริกเกอร์ใหม่

## พารามิเตอร์

- `action` (string, จำเป็น): การดำเนินการที่จะดำเนินการ — ค่าหนึ่งใน `list`, `get`, `create`, `update` หรือ `run`
- `trigger_id` (string, จำเป็นสำหรับ `get`, `update` และ `run`): ตัวระบุของทริกเกอร์ที่จะดำเนินการ ต้องตรงกับรูปแบบ `^[\w-]+$` (เฉพาะอักขระคำและเครื่องหมายยัติภังค์เท่านั้น)
- `body` (object, จำเป็นสำหรับ `create` และ `update`; ไม่บังคับสำหรับ `run`): เพย์โหลดของคำขอที่ส่งไปยัง API

## ตัวอย่าง

### ตัวอย่างที่ 1: แสดงรายการทริกเกอร์ทั้งหมด

```json
{
  "action": "list"
}
```

เรียก `GET /v1/code/triggers` และส่งคืนอาร์เรย์ JSON ของทริกเกอร์ทั้งหมดที่เชื่อมโยงกับบัญชีที่ผ่านการรับรองความถูกต้อง

### ตัวอย่างที่ 2: สร้างทริกเกอร์ใหม่ที่รันทุกเช้าวันทำงาน

```json
{
  "action": "create",
  "body": {
    "name": "weekday-morning-report",
    "schedule": "0 8 * * 1-5",
    "description": "สร้างสรุปรายวันทุกวันทำงานเวลา 08:00 UTC"
  }
}
```

เรียก `POST /v1/code/triggers` พร้อมเนื้อหาที่ให้มาและส่งคืนออบเจ็กต์ทริกเกอร์ที่สร้างใหม่ รวมถึง `trigger_id` ที่กำหนดไว้

### ตัวอย่างที่ 3: เรียกใช้ทริกเกอร์ตามต้องการ

```json
{
  "action": "run",
  "trigger_id": "my-report-trigger"
}
```

เรียก `POST /v1/code/triggers/my-report-trigger/run` ทันที โดยข้ามเวลาที่กำหนดไว้

### ตัวอย่างที่ 4: ดึงข้อมูลทริกเกอร์เดียว

```json
{
  "action": "get",
  "trigger_id": "my-report-trigger"
}
```

เรียก `GET /v1/code/triggers/my-report-trigger` และส่งคืนการกำหนดค่าทริกเกอร์แบบสมบูรณ์

## หมายเหตุ

- โทเค็น OAuth ถูกฉีดเข้าในกระบวนการโดยเครื่องมือ — อย่าคัดลอก วาง หรือบันทึกโทเค็นด้วยตนเองเด็ดขาด การทำเช่นนั้นก่อให้เกิดความเสี่ยงด้านความปลอดภัยและไม่จำเป็นเมื่อใช้เครื่องมือนี้
- ควรใช้เครื่องมือนี้แทน `curl` แบบดิบหรือ HTTP client อื่น ๆ สำหรับการเรียก API ทริกเกอร์ทั้งหมด การใช้ HTTP โดยตรงจะข้ามการฉีดโทเค็นที่ปลอดภัยและอาจเปิดเผยข้อมูลประจำตัว
- เครื่องมือส่งคืนการตอบสนอง JSON แบบดิบจาก API ผู้เรียกมีหน้าที่รับผิดชอบในการแยกวิเคราะห์การตอบสนองและจัดการรหัสสถานะข้อผิดพลาด
- ค่า `trigger_id` ต้องตรงกับรูปแบบ `^[\w-]+$` — อนุญาตเฉพาะอักขระตัวอักษรและตัวเลข เครื่องหมายขีดล่าง และเครื่องหมายยัติภังค์เท่านั้น ช่องว่างหรืออักขระพิเศษจะทำให้คำขอล้มเหลว
