# TeamDelete

Сносит текущую активную команду, удаляя её каталог конфигурации и каталог общих задач. Это очистительный антипод `TeamCreate`, обычно вызываемый после достижения цели команды и завершения работы всеми сокомандниками.

## Когда использовать

- Команда завершила работу и финальный отчёт доставлен пользователю.
- Команда создана по ошибке, либо её область настолько изменилась, что стартовать с чистого листа чище, чем продолжать.
- Нужно создать новую команду, но одна уже активна — сначала удалите старую, поскольку одновременно можно лидировать только одну команду.
- Команда стала застойной между сессиями, и её сохранённое состояние под `~/.claude/teams/` больше не нужно.

НЕ вызывайте, пока сокомандники всё ещё работают — сначала завершите их работу через `SendMessage` с `shutdown_request`, дождитесь каждого `shutdown_response`, затем удаляйте.

## Параметры

`TeamDelete` в своём типичном вызове не принимает параметров. Действует на текущую активную команду, принадлежащую вызывающей сессии.

## Примеры

### Пример 1: штатное завершение работы после успеха

1. Разошлите запрос на завершение команде:
   ```
   SendMessage(to="*", message={ "type": "shutdown_request" })
   ```
2. Дождитесь, пока каждый сокомандник ответит `shutdown_response`.
3. Вызовите `TeamDelete()`, чтобы удалить каталог команды и каталог задач.

### Пример 2: замена неверно сконфигурированной команды

Если `TeamCreate` был вызван с неверным `agent_type` или `description`, сначала убедитесь, что сокомандники ещё не порождены (или заверши их работу), затем:

```
TeamDelete()
TeamCreate(team_name="...", description="...", agent_type="...")
```

## Примечания

- `TeamDelete` падает, если какой-либо сокомандник всё ещё активен. Ответ с ошибкой перечисляет живых сокомандников — отправьте каждому `shutdown_request` через `SendMessage`, дождитесь их `shutdown_response` и повторите.
- Удаление с точки зрения инструмента необратимо. Конфиг команды в `~/.claude/teams/{team_name}/config.json` и её каталог задач удаляются с диска. Если нужен сохранённый список задач, экспортируйте или скопируйте каталог перед удалением.
- Удалить команду может только лидерская сессия, которая её создала. Порождённый сокомандник не может вызвать `TeamDelete` на своей команде.
- Удаление команды не откатывает никакие изменения файловой системы, сделанные сокомандниками в репозитории. Это обычные git-отслеживаемые правки, и их нужно откатывать отдельно, если они нежелательны.
- После успешного возврата `TeamDelete` сессия свободна снова вызвать `TeamCreate` для новой команды.
