# RemoteTrigger

Вызывает API удалённых триггеров claude.ai для управления запланированными задачами и выполнением триггеров по требованию. Токен OAuth обрабатывается инструментом внутренне и никогда не передаётся модели или оболочке.

## Когда использовать

- Управление удалёнными агентами (триггерами) на claude.ai, включая просмотр списка, проверку и обновление существующих триггеров
- Создание новой автоматизированной задачи на основе cron, которая запускает агента Claude по повторяющемуся расписанию
- Запуск существующего триггера по требованию без ожидания следующего запланированного выполнения
- Отображение или аудит всех текущих триггеров для проверки их конфигурации и состояния
- Обновление настроек триггера, таких как расписание, полезная нагрузка или описание, без необходимости его пересоздания

## Параметры

- `action` (string, обязательный): выполняемая операция — одно из значений `list`, `get`, `create`, `update` или `run`
- `trigger_id` (string, обязателен для `get`, `update` и `run`): идентификатор триггера, над которым выполняется операция; должен соответствовать шаблону `^[\w-]+$` (только символы слов и дефисы)
- `body` (object, обязателен для `create` и `update`; необязателен для `run`): полезная нагрузка запроса, отправляемая в API

## Примеры

### Пример 1: получить список всех триггеров

```json
{
  "action": "list"
}
```

Вызывает `GET /v1/code/triggers` и возвращает массив JSON всех триггеров, связанных с аутентифицированным аккаунтом.

### Пример 2: создать новый триггер, запускаемый каждое утро в рабочий день

```json
{
  "action": "create",
  "body": {
    "name": "weekday-morning-report",
    "schedule": "0 8 * * 1-5",
    "description": "Генерировать ежедневную сводку каждый рабочий день в 08:00 UTC"
  }
}
```

Вызывает `POST /v1/code/triggers` с предоставленным телом запроса и возвращает объект нового триггера, включая присвоенный `trigger_id`.

### Пример 3: запустить триггер по требованию

```json
{
  "action": "run",
  "trigger_id": "my-report-trigger"
}
```

Немедленно вызывает `POST /v1/code/triggers/my-report-trigger/run`, минуя запланированное время.

### Пример 4: получить один триггер

```json
{
  "action": "get",
  "trigger_id": "my-report-trigger"
}
```

Вызывает `GET /v1/code/triggers/my-report-trigger` и возвращает полную конфигурацию триггера.

## Примечания

- Токен OAuth внедряется инструментом внутри процесса — никогда не копируйте, не вставляйте и не записывайте токены вручную; это создаёт угрозу безопасности и не нужно при использовании данного инструмента.
- Используйте этот инструмент вместо прямого `curl` или других HTTP-клиентов для всех вызовов API триггеров; прямое использование HTTP обходит безопасное внедрение токена и может раскрыть учётные данные.
- Инструмент возвращает необработанный JSON-ответ от API; вызывающая сторона несёт ответственность за разбор ответа и обработку кодов состояния ошибок.
- Значение `trigger_id` должно соответствовать шаблону `^[\w-]+$` — допускаются только буквенно-цифровые символы, символы подчёркивания и дефисы; пробелы или специальные символы приведут к сбою запроса.
