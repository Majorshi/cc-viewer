# TeamCreate

Учреждает новую команду для совместной работы с общим списком задач и каналом обмена сообщениями между агентами. Команда — примитив координации для многоагентной работы: основная сессия выступает лидером и порождает именованных сокомандников через инструмент `Agent`.

## Когда использовать

- Пользователь явно запрашивает команду, swarm, crew или многоагентное сотрудничество.
- Проект имеет несколько чётко независимых потоков работы, выигрывающих от выделенных специалистов (например, frontend, backend, тесты, документация).
- Нужен постоянный общий список задач, который несколько агентов обновляют по мере прогресса.
- Вам нужны именованные, адресуемые сокомандники, способные обмениваться сообщениями через `SendMessage`, а не одноразовые вызовы подагента.

НЕ используйте для одиночного делегированного поиска или разового параллельного разветвления — обычные вызовы `Agent` легче и достаточны.

## Параметры

- `team_name` (string, обязательный): уникальный идентификатор команды. Используется как имя каталога под `~/.claude/teams/` и как аргумент `team_name` при порождении сокомандников.
- `description` (string, обязательный): краткая формулировка цели команды. Показывается каждому сокомандник при старте и записывается в конфиг команды.
- `agent_type` (string, необязательный): тип персоны подагента по умолчанию, применяемый к сокомандникам, не переопределяющим его. Типичные значения: `general-purpose`, `Explore` или `Plan`.

## Примеры

### Пример 1: создание команды рефакторинга

```
TeamCreate(
  team_name="refactor-crew",
  description="Refactor the data access layer from raw SQL to Prisma, including migrations and tests.",
  agent_type="general-purpose"
)
```

После создания порождайте сокомандников через `Agent` с `team_name: "refactor-crew"` и разными значениями `name`, такими как `db-lead`, `migrations` и `tests`.

### Пример 2: создание исследовательской команды

```
TeamCreate(
  team_name="perf-investigation",
  description="Identify and rank the top three performance regressions introduced in the last release.",
  agent_type="Explore"
)
```

Каждый порождённый сокомандник наследует `Explore` как персону по умолчанию, соответствующую исследовательскому характеру работы только на чтение.

## Примечания

- Из одной сессии можно лидировать только одну команду одновременно. Завершите или удалите текущую команду перед созданием новой.
- Команда связана 1:1 с общим списком задач. Лидер владеет созданием, назначением и закрытием задач; сокомандники обновляют статус задач, над которыми работают.
- Конфигурация команды сохраняется в `~/.claude/teams/{team_name}/config.json`, а каталог задач живёт рядом с ней. Эти файлы переживают сессии до явного удаления через `TeamDelete`.
- Сокомандники порождаются инструментом `Agent` с совпадающим `team_name` плюс уникальным `name`. `name` становится адресом, используемым `SendMessage`.
- Выбирайте `team_name`, безопасный для файловой системы (буквы, цифры, дефисы, подчёркивания). Избегайте пробелов или слэшей.
- Пишите `description` так, чтобы совершенно новый сокомандник, читая его с чистого листа, понимал цель команды без дополнительного контекста. Оно становится частью стартового промпта каждого сокомандника.
