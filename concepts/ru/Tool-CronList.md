# CronList

Выводит список всех задач cron, запланированных с помощью `CronCreate` в текущем сеансе. Возвращает сводку по каждой активной задаче, включая `id`, выражение cron, сокращённый `prompt`, флаг `recurring`, флаг `durable` и время следующего запуска.

## Когда использовать

- Для проверки всех текущих запланированных задач перед внесением изменений или завершением сеанса.
- Для нахождения корректного `id` задачи перед вызовом `CronDelete` с целью её удаления.
- Для диагностики причин, по которым ожидаемая задача не была запущена: проверить её существование и время следующего запуска.
- Для подтверждения того, что одноразовая (неповторяющаяся) задача ещё не была выполнена и по-прежнему ожидает запуска.

## Параметры

Нет.

## Примеры

### Пример 1: просмотр всех запланированных задач

Вызвать `CronList` без аргументов, чтобы получить полный список всех активных задач cron. В ответе для каждой задачи содержится `id`, выражение cron, определяющее расписание, сокращённая версия `prompt`, который будет выполнен, признак `recurring` (повторяющаяся), признак `durable` (сохраняется между перезапусками) и следующее запланированное время запуска.

### Пример 2: поиск id конкретной повторяющейся задачи

Если было создано несколько задач cron и необходимо удалить одну конкретную повторяющуюся задачу, сначала вызвать `CronList`. Просмотреть возвращённый список и найти задачу, сводка `prompt` и выражение cron которой совпадают с целевой задачей. Скопировать её `id` и передать в `CronDelete`.

## Примечания

- В списке отображаются только задачи, созданные в текущем сеансе, если только они не были созданы с `durable: true`, что позволяет им сохраняться между перезапусками сеанса.
- Поле `prompt` в сводке усечено: отображается только начало полного текста запроса, а не всё его содержимое.
- Одноразовые задачи (`recurring` равно `false`), которые уже были выполнены, автоматически удаляются и больше не отображаются в списке.
