# CronDelete

Отменяет задание cron, ранее запланированное с помощью `CronCreate`. Немедленно удаляет его из сеансового хранилища в памяти. Не оказывает никакого эффекта, если задание уже было автоматически удалено (однократные задания удаляются после срабатывания, повторяющиеся задания истекают через 7 дней).

## Когда использовать

- Пользователь просит остановить повторяющееся запланированное задание до автоматического истечения срока через 7 дней.
- Однократное задание больше не нужно и должно быть отменено до его срабатывания.
- Необходимо изменить выражение расписания существующего задания — удалите его с помощью `CronDelete`, затем создайте заново с помощью `CronCreate`, используя новое выражение.
- Очистка нескольких устаревших заданий для поддержания порядка в сеансовом хранилище.

## Параметры

- `id` (string, обязательный): Идентификатор задания, возвращённый `CronCreate` при первоначальном создании задания. Это значение должно точно совпадать; нечёткий поиск или поиск по имени не поддерживается.

## Примеры

### Пример 1: отмена работающего повторяющегося задания

Повторяющееся задание было ранее создано с идентификатором `"cron_abc123"`. Пользователь просит его остановить.

```
CronDelete({ id: "cron_abc123" })
```

Задание удаляется из сеансового хранилища и больше не будет срабатывать.

### Пример 2: удаление устаревшего однократного задания до его срабатывания

Однократное задание с идентификатором `"cron_xyz789"` было запланировано к выполнению через 30 минут, но пользователь решил, что оно больше не нужно.

```
CronDelete({ id: "cron_xyz789" })
```

Задание отменяется. Никаких действий не будет выполнено, когда наступит первоначальное время срабатывания.

## Примечания

- `id` должен быть получен из возвращаемого значения `CronCreate`. Нет способа найти задание по описанию или обратному вызову — сохраните идентификатор, если может потребоваться отменить задание позже.
- Если задание уже было автоматически удалено (сработало как однократное или достигло повторяющегося истечения срока через 7 дней), вызов `CronDelete` с этим идентификатором является холостой операцией и не вызовет ошибки.
- `CronDelete` влияет только на текущий сеанс в памяти. Если среда выполнения не сохраняет состояние cron между перезапусками, запланированные задания будут потеряны при перезапуске независимо от того, был ли вызван `CronDelete`.
- Операции массового удаления не существует; отменяйте каждое задание по отдельности, используя его собственный `id`.
