# executeCode

Выполняет фрагмент кода внутри живого ядра или песочницы, предоставляемых IDE-интеграцией (например, ядра Jupyter, привязанного к открытому блокноту). Инструмент присутствует только тогда, когда Claude Code запущен вместе с совместимым IDE-мостом, таким как расширение VS Code с выбранным ядром Jupyter.

## Когда использовать

- Запуск быстрого вычисления, инспекции данных или графика против состояния, уже загруженного в активном ядре Jupyter.
- Валидация фрагмента кода перед вставкой в ячейку блокнота.
- Исследование dataframe, переменной или модели в памяти, которая существует в ядре, но не сериализована на диск.
- Производство графика или численного результата, который пользователь хочет отрендерить inline в IDE.

НЕ используйте для отдельных скриптов, которым лучше подошёл бы `Bash` с запуском `python script.py`, или для кода, который должен пережить свежее ядро.

## Параметры

- `code` (string, обязательный): код для выполнения в текущем ядре. Запускается так, будто вставлен в ячейку блокнота — определённые переменные сохраняются в ядре до его перезапуска.
- `language` (string, необязательный): язык фрагмента, когда IDE-мост поддерживает несколько ядер. Чаще всего опускается; по умолчанию — язык активного ядра (обычно Python).

## Примеры

### Пример 1: инспекция dataframe в памяти

```
executeCode(
  code="df.head()\nprint(df.shape)\nprint(df.dtypes)"
)
```

Возвращает первые строки, форму и типы колонок dataframe, уже загруженного в ядре.

### Пример 2: быстрая численная проверка

```
executeCode(
  code="import numpy as np\nnp.mean([1, 2, 3, 4, 5])"
)
```

Выполняет разовое вычисление без создания ячейки блокнота.

## Примечания

- `executeCode` — инструмент IDE-моста. Он недоступен в обычных терминальных сессиях Claude Code; появляется только когда сессия подключена к IDE, экспонирующей ядро (например, расширение VS Code Jupyter).
- Состояние сохраняется в ядре. Переменные, определённые одним вызовом `executeCode`, остаются видимы последующим вызовам, ячейкам блокнота и пользователю — помните о побочных эффектах.
- Долго идущий или блокирующий код блокирует ядро. Держите фрагменты короткими; для многоминутной работы напишите настоящий скрипт и запускайте через `Bash`.
- Вывод (stdout, возвращаемые значения, изображения) возвращается в сессию. Очень большие выводы могут усекаться IDE-мостом.
- Для правок файлов предпочитайте `Edit`, `Write` или `NotebookEdit` — `executeCode` не заменяет авторинг исходных файлов.
