# CronList

현재 세션에서 `CronCreate`를 통해 예약된 모든 크론 작업을 나열합니다. 각 활성 크론의 `id`, cron 표현식, 축약된 `prompt`, `recurring` 플래그, `durable` 플래그, 다음 실행 시간을 포함한 요약을 반환합니다.

## 사용 시점

- 변경을 가하거나 세션을 종료하기 전에 현재 예약된 작업을 감사합니다.
- `CronDelete`를 호출하여 작업을 삭제하기 전에 올바른 `id`를 찾습니다.
- 작업의 존재 여부와 다음 실행 시간을 확인하여 예상된 작업이 실행되지 않은 원인을 디버그합니다.
- 일회성(비반복) 작업이 아직 실행되지 않고 대기 중인지 확인합니다.

## 매개변수

없음.

## 예시

### 예시 1: 예약된 모든 작업 감사

인수 없이 `CronList`를 호출하면 모든 활성 크론 작업의 전체 목록을 가져올 수 있습니다. 응답에는 각 작업의 `id`, cron 표현식, `prompt`의 축약 버전, `recurring`(반복) 여부, `durable`(영속화) 여부, 다음 예약 실행 시간이 포함됩니다.

### 예시 2: 특정 반복 작업의 id 찾기

여러 크론 작업을 생성했고 그 중 하나를 삭제해야 한다면, 먼저 `CronList`를 호출하세요. 반환된 목록에서 `prompt` 요약과 cron 표현식이 삭제하려는 작업과 일치하는 항목을 찾아 해당 `id`를 복사하여 `CronDelete`에 전달합니다.

## 참고사항

- `durable: true`로 생성된 작업을 제외하고, 현재 세션에서 생성된 작업만 나열됩니다. `durable: true` 작업은 세션 재시작 후에도 유지됩니다.
- 요약의 `prompt` 필드는 잘려 있으며, 전체 프롬프트 텍스트의 앞부분만 표시됩니다.
- 이미 실행된 일회성 작업(`recurring`이 `false`)은 자동으로 삭제되어 목록에 더 이상 표시되지 않습니다.
