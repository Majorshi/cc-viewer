# getDiagnostics

연결된 IDE에서 특정 파일이나 IDE가 현재 열어 놓은 모든 파일에 대한 언어 서버 진단 (오류, 경고, 힌트)을 가져옵니다. 작업을 완료했다고 선언하기 전에 코드 변경이 깔끔하게 컴파일되는지 확인하는 데 사용됩니다.

## 사용 시점

- `Edit`나 `Write` 후 변경으로 타입 오류, 구문 오류, 또는 린트 경고가 발생하지 않았는지 확인.
- 작업을 종료하기 전에 미해결 문제를 위해 모든 열린 파일 스위핑.
- 사용자가 보고한 오류를 진단할 때 — IDE에서 정확한 컴파일러나 타입 체커 메시지를 가져오면 추측을 피할 수 있습니다.
- 정적 정확성만 확인해야 할 때 전체 빌드나 테스트 명령 실행의 가벼운 대안으로.

테스트 스위트의 대체물로 `getDiagnostics`에 의존하지 마십시오. 언어 서버가 보는 것을 보고하며 테스트나 프로덕션 시에 실행되는 것을 보고하지는 않습니다.

## 매개변수

- `uri` (string, 선택): 진단을 가져올 파일 URI (일반적으로 `file:///absolute/path`). 생략되면 도구는 IDE가 현재 열고 있는 모든 파일에 대한 진단을 반환합니다.

## 예시

### 예시 1: 편집 후 단일 파일 확인

```
getDiagnostics(
  uri="file:///Users/sky/project/src/auth.ts"
)
```

`src/auth.ts`에 대한 TypeScript 오류, ESLint 경고, 또는 기타 언어 서버 메시지를 반환합니다.

### 예시 2: 모든 열린 파일 스위핑

```
getDiagnostics()
```

현재 열려 있는 모든 편집기에서 진단을 반환합니다. 다중 파일 리팩터링의 끝에서 다른 곳에서 회귀가 없는지 확인하는 데 유용합니다.

## 참고사항

- `getDiagnostics`는 IDE 브리지 도구입니다. Claude Code가 지원 IDE 통합 (예: VS Code 확장)에 연결된 경우에만 사용할 수 있습니다. 일반 터미널 세션에서는 도구가 나타나지 않습니다.
- 결과는 IDE가 로드한 모든 언어 서버 — TypeScript, Pyright, ESLint, rust-analyzer 등을 반영합니다. 품질과 범위는 Claude Code가 아닌 사용자의 IDE 설정에 따라 달라집니다.
- 진단은 라이브입니다. 편집 후 언어 서버가 다시 분석할 시간을 주기 전에 빈 결과를 성공으로 해석하지 마십시오 — 파일이 방금 저장된 경우 다시 실행하십시오.
- 심각도 수준에는 일반적으로 `error`, `warning`, `information`, `hint`가 포함됩니다. 먼저 `error`에 집중하십시오. 경고는 의도적인 프로젝트 스타일일 수 있습니다.
- IDE에 현재 열려 있지 않은 파일의 경우 파일에 실제 문제가 있어도 언어 서버에 보고할 진단이 없을 수 있습니다. 권위 있는 범위를 위해 파일을 열거나 빌드를 실행하십시오.
