# executeCode

IDE 통합 (예: 현재 열려 있는 노트북에 바인딩된 Jupyter 커널)이 제공하는 라이브 커널이나 샌드박스 내에서 코드 스니펫을 실행합니다. 이 도구는 Jupyter 커널이 선택된 VS Code 확장과 같은 호환 IDE 브리지와 함께 Claude Code가 실행될 때만 존재합니다.

## 사용 시점

- 활성 Jupyter 커널에 이미 로드된 상태에 대해 빠른 계산, 데이터 검사, 또는 플롯 실행.
- 노트북 셀에 붙여 넣기 전에 코드 스니펫 검증.
- 커널에 존재하지만 디스크에 직렬화되지 않은 메모리 내 데이터프레임, 변수, 또는 모델 탐색.
- 사용자가 IDE에 인라인으로 렌더링하기를 원하는 차트나 숫자 결과 생성.

`Bash`가 `python script.py`를 실행하는 것이 더 나은 독립 스크립트나, 새로운 커널을 넘어 지속되어야 하는 코드에는 사용하지 마십시오.

## 매개변수

- `code` (string, 필수): 현재 커널에서 실행할 코드. 노트북 셀에 붙여 넣은 것처럼 실행됩니다 — 정의된 변수는 재시작될 때까지 커널에 지속됩니다.
- `language` (string, 선택): IDE 브리지가 여러 커널을 지원할 때 스니펫의 언어. 가장 일반적으로 생략됩니다. 기본값은 활성 커널의 언어 (일반적으로 Python)입니다.

## 예시

### 예시 1: 메모리 내 데이터프레임 검사

```
executeCode(
  code="df.head()\nprint(df.shape)\nprint(df.dtypes)"
)
```

커널에 이미 로드된 데이터프레임의 첫 번째 행, 모양, 열 dtypes를 반환합니다.

### 예시 2: 빠른 숫자 확인

```
executeCode(
  code="import numpy as np\nnp.mean([1, 2, 3, 4, 5])"
)
```

노트북 셀을 생성하지 않고 일회성 계산을 실행합니다.

## 참고사항

- `executeCode`는 IDE 브리지 도구입니다. Claude Code의 일반 터미널 세션에서는 사용할 수 없습니다. 커널을 노출하는 IDE (예: VS Code Jupyter 확장)에 세션이 연결된 경우에만 나타납니다.
- 상태는 커널에 지속됩니다. 한 `executeCode` 호출에서 정의된 변수는 나중 호출, 노트북 셀, 사용자에게 여전히 보입니다 — 부작용에 유의하십시오.
- 장시간 실행되거나 차단되는 코드는 커널을 차단합니다. 스니펫을 짧게 유지하십시오. 수 분 작업의 경우 실제 스크립트를 작성하고 `Bash`를 통해 실행하십시오.
- 출력 (stdout, 반환 값, 이미지)은 세션에 반환됩니다. 매우 큰 출력은 IDE 브리지에 의해 잘릴 수 있습니다.
- 파일 편집의 경우 `Edit`, `Write`, 또는 `NotebookEdit`를 선호하십시오 — `executeCode`는 소스 파일 작성의 대체물이 아닙니다.
