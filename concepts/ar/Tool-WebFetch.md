# WebFetch

يسترجع محتويات صفحة ويب عامة، ويحول HTML إلى Markdown، ويُشغِّل نموذجاً مساعداً صغيراً على النتيجة باستخدام موجه بلغة طبيعية لاستخراج المعلومات التي تحتاجها.

## متى يُستخدم

- قراءة صفحة توثيق عامة أو منشور مدونة أو RFC مُشار إليه في المحادثة.
- استخراج حقيقة محددة أو مقتطف شفرة أو جدول من URL معروف دون تحميل الصفحة كاملة في السياق.
- تلخيص ملاحظات الإصدار أو سجلات التغييرات من مورد ويب مفتوح.
- التحقق من مرجع API العام لمكتبة عندما لا يكون المصدر في المستودع المحلي.
- اتباع رابط لصقه المستخدم في المحادثة للإجابة عن سؤال متابعة.

## المعاملات

- `url` (سلسلة، مطلوب): URL مطلق مُشكَّل بالكامل. يُرقى `http://` العادي تلقائياً إلى `https://`.
- `prompt` (سلسلة، مطلوب): التعليمة المُمرَّرة إلى نموذج الاستخراج الصغير. صف بدقة ما يُستخرج من الصفحة، مثل «list all exported functions» أو «return the minimum supported Node version».

## أمثلة

### مثال 1: استخراج افتراض إعداد

```
WebFetch(
  url="https://vitejs.dev/config/server-options.html",
  prompt="What is the default value of server.port and can it be a string?"
)
```

تجلب الأداة صفحة توثيق Vite، وتحولها إلى Markdown، وتُعيد إجابة موجزة مثل "Default is `5173`; accepts a number only."

### مثال 2: تلخيص قسم سجل تغييرات

```
WebFetch(
  url="https://nodejs.org/en/blog/release/v20.11.0",
  prompt="List the security fixes included in this release as bullet points."
)
```

مفيد عندما يسأل المستخدم «ما الذي تغير في Node 20.11» وصفحة الإصدار طويلة.

## ملاحظات

- `WebFetch` يفشل على أي URL يتطلب مصادقة أو ملفات تعريف ارتباط أو VPN. لـ Google Docs أو Confluence أو Jira أو موارد GitHub الخاصة أو الويكيات الداخلية، استخدم خادم MCP مخصصاً يوفر وصولاً مصادقاً عليه بدلاً من ذلك.
- لأي شيء مستضاف على GitHub (PRs، issues، ملفات blob، استجابات API)، فضِّل CLI `gh` عبر `Bash` على كشط واجهة الويب. `gh pr view` و`gh issue view` و`gh api` تُعيد بيانات منظمة وتعمل ضد المستودعات الخاصة.
- قد تُلخَّص النتائج عندما تكون الصفحة المجلوبة كبيرة جداً. إذا كنت بحاجة إلى نص دقيق، ضيِّق `prompt` ليطلب استخراجاً حرفياً.
- يُطبَّق كاش ذاتي التنظيف لمدة 15 دقيقة لكل URL. الاستدعاءات المتكررة لنفس الصفحة خلال جلسة واحدة تكون شبه فورية لكن قد تُعيد محتوى قديماً قليلاً. إذا كانت الحداثة مهمة، اذكر ذلك في الموجه أو انتظر الكاش.
- إذا أصدر المضيف المستهدف إعادة توجيه عبر المضيفين، تُعيد الأداة URL الجديد في كتلة استجابة خاصة ولا تتبعه تلقائياً. استدعِ `WebFetch` مرة أخرى بهدف إعادة التوجيه إذا كنت لا تزال تريد المحتوى.
- يُنفذ الموجه نموذج أصغر وأسرع من المساعد الرئيسي. أبقِه ضيقاً ومحدداً؛ يُعالَج الاستدلال المعقد متعدد الخطوات أفضل بقراءة Markdown الخام بنفسك بعد الجلب.
- لا تمرر أبداً أسراراً أو رموزاً أو معرفات جلسة مضمنة في URL — قد تُسجل محتويات الصفحة وسلاسل الاستعلام المنعكسة في المخرجات بواسطة الخدمات الأعلى.
