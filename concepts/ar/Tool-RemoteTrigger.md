# RemoteTrigger

يستدعي واجهة برمجة تطبيقات المشغّلات البعيدة في claude.ai لإدارة المهام المجدولة وتنفيذ المشغّلات عند الطلب. يتم التعامل مع رمز OAuth داخليًا بواسطة الأداة ولا يتعرض أبدًا للنموذج أو الواجهة البرمجية للسطر.

## متى يُستخدم

- إدارة العوامل البعيدة (المشغّلات) على claude.ai، بما في ذلك سرد المشغّلات الموجودة وفحصها وتحديثها
- إنشاء مهمة آلية جديدة قائمة على cron تُشغّل عامل Claude وفق جدول زمني متكرر
- تشغيل مشغّل موجود عند الطلب دون انتظار تشغيله المجدول التالي
- سرد أو مراجعة جميع المشغّلات الحالية للاطلاع على إعداداتها وحالتها
- تحديث إعدادات المشغّل كالجدول الزمني أو الحمولة أو الوصف دون الحاجة إلى إعادة إنشائه

## المعاملات

- `action` (string، مطلوب): العملية المراد تنفيذها — إحدى القيم `list` أو `get` أو `create` أو `update` أو `run`
- `trigger_id` (string، مطلوب لـ `get` و`update` و`run`): معرّف المشغّل المراد تنفيذ العملية عليه؛ يجب أن يطابق النمط `^[\w-]+$` (أحرف الكلمات والشرطات فقط)
- `body` (object، مطلوب لـ `create` و`update`؛ اختياري لـ `run`): حمولة الطلب المرسلة إلى الواجهة البرمجية

## أمثلة

### مثال 1: سرد جميع المشغّلات

```json
{
  "action": "list"
}
```

يستدعي `GET /v1/code/triggers` ويُعيد مصفوفة JSON بجميع المشغّلات المرتبطة بالحساب المصادق عليه.

### مثال 2: إنشاء مشغّل جديد يعمل كل صباح في أيام الأسبوع

```json
{
  "action": "create",
  "body": {
    "name": "weekday-morning-report",
    "schedule": "0 8 * * 1-5",
    "description": "إنشاء ملخص يومي في كل يوم عمل الساعة 08:00 UTC"
  }
}
```

يستدعي `POST /v1/code/triggers` مع الجسم المقدَّم ويُعيد كائن المشغّل المنشأ حديثًا متضمنًا `trigger_id` المخصص له.

### مثال 3: تشغيل مشغّل عند الطلب

```json
{
  "action": "run",
  "trigger_id": "my-report-trigger"
}
```

يستدعي فورًا `POST /v1/code/triggers/my-report-trigger/run`، متجاوزًا الوقت المجدول.

### مثال 4: جلب مشغّل واحد

```json
{
  "action": "get",
  "trigger_id": "my-report-trigger"
}
```

يستدعي `GET /v1/code/triggers/my-report-trigger` ويُعيد الإعداد الكامل للمشغّل.

## ملاحظات

- يتم حقن رمز OAuth داخل العملية بواسطة الأداة — لا تقم أبدًا بنسخ الرموز أو لصقها أو تسجيلها يدويًا؛ فذلك يُشكّل خطرًا أمنيًا وهو أمر غير ضروري عند استخدام هذه الأداة.
- يُفضَّل استخدام هذه الأداة بدلًا من `curl` المباشر أو غيره من عملاء HTTP لجميع استدعاءات واجهة برمجة تطبيقات المشغّلات؛ إذ إن الاستخدام المباشر لـ HTTP يتجاوز الحقن الآمن للرمز وقد يكشف بيانات الاعتماد.
- تُعيد الأداة استجابة JSON الخام من الواجهة البرمجية؛ ويتحمل المُستدعي مسؤولية تحليل الاستجابة والتعامل مع رموز حالة الأخطاء.
- يجب أن تتطابق قيمة `trigger_id` مع النمط `^[\w-]+$` — يُسمح فقط بالأحرف الأبجدية الرقمية والشرطات السفلية والشرطات؛ وستؤدي المسافات أو الأحرف الخاصة إلى فشل الطلب.
